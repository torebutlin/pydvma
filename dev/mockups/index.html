pydvma — Stage 2 GUI mockups, round 1

Three layout topologies, one shared design language. Compare the navigation model, not the polish — everything here is throwaway HTML.

Option A

Refined three-pane — "the familiar, made slick"

Today's geography (views left · plot centre · tools right), modernised: accordion tool sections replace the 11-entry dropdown, rails collapse, scope docks as a strip under the plot.

 Option B

Workflow bench — "the guided flow"

A stage ribbon across the top (Setup → Acquire → Time → Frequency → TF → Fit → Export) sets both the view and a slim context tool strip. A first-class data tray of set-cards on the left owns navigation and comparison.

 Option C

Instrument rack — "the DAW take"

Recording-studio mental model: transport bar (record, pretrigger arm, output signal chip, master levels), a channel rack with per-line strips, the plot as a stage, and processing modules (FFT / TF / Sono / Fit) as rack units.
Things to try in each one:
  • Press Log Data / the record button — watch the fake capture land as a new set.
  • Click line colour chips to cycle on / fade / off; double-click a label to rename.
  • Switch views (Time / Frequency / TF / Sonogram) and drag the N-frames slider in the TF view.
  • Open the scope/monitor and note where it lives; spot the clipping indicator.
  • Hit the dashed Narrow toggle (top-right) to preview the half-screen adaptive state.
